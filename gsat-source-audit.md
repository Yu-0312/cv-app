## 目前本地資料庫現況

- 現有 `index.html` 內建 `DEPTS` 歷年落點資料約 50 筆校系組合。
- 涵蓋學校 17 間，離「全台大學、全校系可比對」還有很大缺口。
- `university-data.js` 已補齊教育部官方大學與學士班科系名單，現在缺的是「校系落點分數 / 風險區間」資料。

## 盤點原則

- 只看是否能取得「校系層級」的落點結果，不只看首頁是否存在。
- 優先評估是否有公開 API、可讀結構化資料、或穩定頁面可整理。
- 若來源需要登入、付費、准考證綁定、或有強防爬，標示為不適合直接自動匯入。

## 目前最值得優先投入的來源

1. `104 落點分析`
   - 前端腳本已露出明確 API 路徑：
     - `https://student.104.com.tw/api/v1.0/hs/standardScore?scoreYear=115`
     - `https://student.104.com.tw/api/v1.0/ast/tempScore`
     - 前端 JS 也出現 `hs/landingPoint`、`hs/majorList`、`ast/majorList`
   - 代表它很可能不是「沒有資料」，而是需要正確請求格式、瀏覽器狀態、或互動流程。
   - 如果要做真正可維護的匯入器，104 是第一優先。

2. `1111 落點分析`
   - 公開頁面與前端資源可讀。
   - 首頁 HTML 指向 `exam-match-api.1111.com.tw`。
   - 下一步適合用瀏覽器 DevTools / 自動化去抓 XHR。

3. `CEIP`
   - 公開頁確定有學測個申落點頁，適合當結果比對源。
   - 但目前從靜態 HTML 還沒直接看到簡單可重用 API。

## 不建議作為主匯入源的來源

- `甄戰`：准考證綁定、付費 / 會員差異資料。
- `儒林交叉查榜`：Cloudflare 擋機器請求。
- `大學生甘單`：App 型資料，不利目前這個網頁專案直接批量整理。
- `東海大學入口頁`：本身不是完整資料供應端。

## 建議的實作順序

1. 先保留教育部官方 `university-data.js` 當「學校 / 科系主鍵表」。
2. 先研究 `104` 的實際查詢流程，確認是否能穩定取回校系列表與歷年低標 / 預估落點欄位。
3. 再補 `1111` 與 `CEIP` 當交叉驗證來源。
4. 對於登入、付費、防爬站，只拿來人工抽樣校正，不直接寫成自動匯入程序。

## 這輪已確認的具體事實

- `104` 的 `standardScore` API 可直接回傳 115 / 114 年五標。

- `儒林交叉查榜` 直接請求會收到 Cloudflare challenge。
- `CEIP` 的學測落點頁可公開開啟，且頁面上可直接看到各科 0–15 級分輸入介面。
- `東海大學落點分析` 實際是導流到 `aicareer.org.tw`。
